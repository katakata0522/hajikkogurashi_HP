# ミニゲーム企画書：超絶！仕分け工場（仮）

## 1. コンセプト・概要
* **ジャンル**: 耐久・脳トレ・パニックアクション
* **ターゲット**: 集中して無心になりたい人、自分の限界（脳の処理速度）に挑戦したい人
* **コンセプト**: 上から次々と降ってくるアイテムを、瞬時の判断で正しい箱に振り分ける。スピードが上がるにつれて脳がバグっていく「パニック感」と、ゾーンに入った時の「全能感」を楽しむゲーム。

## 2. ゲームのルールと操作方法
* **基本操作**: 画面の左右タップ（スマホ）、またはキーボードの左右矢印キー（PC）。
* **プレイルーム**:
  1. **アイテムの落下**: 画面中央の上部から、コンベアに乗るようにアイテムが1つずつ落ちてきます。
  2. **仕分け（スワイプ/フリック/タップ）**: 
     * 落ちてきたアイテムが「Aの条件」なら左の箱へ、「Bの条件」なら右の箱へ振り分けます。
     * 例：赤いアイテムは左、青いアイテムは右。丸いアイテムは左、四角いアイテムは右など。
  3. **難易度の上昇（脳トレ要素）**:
     * 最初は「赤」か「青」だけ。
     * 途中から「赤い四角」や「青い丸」など、色と形が混ざった引っかけアイテムが登場し、プレイヤーの判断を鈍らせます。
     * 時間経過とともに落下スピードがどんどん上がっていきます。
  4. **ゲームオーバー**:
     * 間違った箱に入れてしまうか、仕分けが間に合わずにアイテムがコンベアの端から落ちてしまうと、即ゲームオーバー（またはライフ減少）。
  5. **リザルト**:
     * 正しく仕分けられた「連続コンボ数（個数）」がそのままスコアになります。

## 3. 画面構成（UI/UX）
* **画面レイアウト**: 縦画面推奨（スマホ操作に最適化）。PCでも中央に縦長のプレイエリアを表示。
* **背景**: 工場のベルトコンベアや、サイバーなデータ処理空間をイメージしたデザイン。
* **UI要素**:
  * 現在のスコア（仕分け数）を画面上部に大きく表示。
  * 画面下部の左右に「仕分け先の箱（またはシュート）」。
  * プレイ中：ミスした時に画面が一瞬赤く光るなどのフィードバック。
  * リザルト画面：
    * 最終スコア（処理したアイテム数）
    * 称号（例：「新人バイト」「工場長」「処理能力：スーパーコンピューター級」）
    * 「もう一度遊ぶ」ボタン
    * 「X（Twitter）でシェア」ボタン

## 4. 演出・デザイン方針
* **デザインテイスト**: 
  * 「ギリギリ・ブレーキ！」のサイバー感と合わせる、あるいは「レトロなドット絵の工場」にして可愛さを出すか、開発時に調整。
  * 視認性が最重要（一瞬で色や形を判断するため、アイテムのデザインは極力シンプルでくっきりしたものにする）。
* **サウンド**:
  * 仕分け成功音（ポポン！という小気味よい音。連続で成功すると音階が上がっていくと気持ちいい）。
  * ミス音（ブーッ！というエラー音）。
  * BGM：徐々にテンポが速くなるテクノ系または作業BGM。

## 5. 拡散（バズ）の仕組み
* **X（Twitter）シェアテキスト案**:
  * 「脳の処理限界に到達…！ 【150個】のアイテムを仕分けました！ 称号：[スーパー工場長]」 ＋ ゲームURL ＋ #はじっこぐらし #超絶仕分け工場

## 6. 技術スタック・実装方針
* **言語**: HTML5 Canvas または DOM要素（CSS Animation）, Vanilla JavaScript
  ※アイテムの数が多くなければ、DOM要素（`<div>`）を上から下に降らせるアニメーションでも実装可能です。今回はスマホでの滑らかさを重視し、Canvasをメインに検討します。
* **ディレクトリ構成予定**:
  ```text
  sorting-factory/
  ├── index.html       (ゲーム画面)
  ├── style.css        (UIのスタイリング)
  └── script.js        (ゲームロジック・生成・判定処理)
  ```
