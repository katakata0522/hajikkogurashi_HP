# ミニゲーム企画書：ギリギリ・ブレーキ！（仮）

## 1. コンセプト・概要
* **ジャンル**: チキンレース・ワンタップアクション
* **ターゲット**: ちょっとした隙間時間に遊びたい人、スコアをSNSで自慢したい人
* **コンセプト**: 「あとちょっと…！」という度胸とタイミングを試すゲーム。猛スピードで走るキャラクター（または乗り物）を、崖の端っこギリギリで止める快感と焦燥感を味わう。

## 2. ゲームのルールと操作方法
* **基本操作**: 画面のどこかを**ワンタップ（またはクリック）するだけ**。
* **プレイルーム**:
  1. **スタート**: 画面をタップすると、キャラクターが左から右へ向かって猛スピードで走り出します。
  2. **ブレーキ**: もう一度画面をタップするとブレーキがかかります。
  3. **滑走**: ブレーキをかけてもすぐには止まらず、慣性でしばらく滑ります（※ここがミソです）。
  4. **リザルト**:
     * **成功**: 崖から落ちずに止まれた場合。崖の端からの距離（例：`残り0.12m！`）がスコアになります。ゼロに近いほど高評価。
     * **失敗（ゲームオーバー）**: ブレーキが遅れて崖から落ちてしまう、または手前すぎる（例：`残り50m以上`など、チキンすぎる）場合。

## 3. 画面構成（UI/UX）
* **画面レイアウト**: 横画面（スマホの場合は縦でも横でも遊べるレスポンシブなCanvas）
* **背景**: 単純な風景（青空、雲、地面、そして右側に断崖絶壁）。
* **キャラクター**: シンプルなドット絵や図形（車、自転車、あるいは走る棒人間など）。
* **UI要素**:
  * タイトルロゴ、スタートボタン（「タップして発進！」）
  * プレイ中：現在のスピードメーター（飾りとしてあると焦燥感が出る）
  * リザルト画面：
    * 崖までの残り距離（スコア）
    * 評価コメント（例：「神業！」「チキン野郎」「あぁぁぁ落下…」）
    * 「もう一度遊ぶ」ボタン
    * 「X（Twitter）でシェア」ボタン

## 4. 演出・演出効果
* **サウンド**:
  * 発進音（ギュオーン！）
  * ブレーキ音（キキーッ！）
  * 成功音（ファンファーレ、または歓声）
  * 落下音（ヒューー……ドスッ）
* **視覚効果**:
  * ブレーキ時の摩擦エフェクト（タイヤの煙など）
  * 崖ギリギリで止まった時の冷や汗エフェクトや、カメラのズームイン演出（余裕があれば）

## 5. 拡散（バズ）の仕組み
* **X（Twitter）シェアテキスト案**:
  * 【成功時】「崖っぷちでストップ！残り距離 0.05m！！ 神回避しました」 ＋ ゲームURL ＋ #はじっこぐらし #ギリギリブレーキ
  * 【失敗時】「あああああ！崖から落ちた…（チキン度 100%）」 ＋ ゲームURL ＋ #はじっこぐらし #ギリギリブレーキ

## 6. 技術スタック・実装方針
* **言語**: HTML5 Canvas, Vanilla JavaScript
* **物理演算**: 自前でシンプルな慣性（速度・加速度・摩擦係数）の計算を実装。Matter.jsなどの外部ライブラリは不要なレベル。
* **ディレクトリ構成予定**:
  ```text
  girigiri-brake/
  ├── index.html       (ゲーム画面・キャンバス)
  ├── style.css        (UIのスタイリング)
  └── script.js        (ゲームロジック・描画処理)
  ```
